# ADR Template

Status: (proposed | accepted | superseded | deprecated)
Date: YYYY-MM-DD

## Context
Describe the motivating factors, background, and problem statement. Provide any constraints, assumptions, and forces (e.g., performance, scalability, cost, team skill).

## Decision
State the decision succinctly. Include chosen option and high-level design implications.

## Alternatives Considered
- Option A: summary + pros/cons
- Option B: summary + pros/cons
- Option C: summary + pros/cons
(Explain why rejected.)

## Consequences
Positive, negative, trade-offs, and follow-up actions or migrations. List measurable impact areas (latency, throughput, cost, complexity, operability, security, compliance).

## Implementation Notes
Key steps, rollout strategy, backward compatibility, migration plan, related cleanup.

## Validation & Metrics
How will success be measured? KPIs, guardrails, monitoring & alerting updates.

## Open Questions
List any unresolved issues or risks requiring follow-up.

## References
Links to specs, tickets, HLD/LLD docs, prior ADRs.
